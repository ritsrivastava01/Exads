Settings Page -View

Problem

When component has a blank path, path: ' ', component: SettingsComponent then, angular loads it as a default component of the application, and when we click on any tab (eg API-Keys), 'Angular active route' activates the parent and child route at same time. Therefore, angular loads full path (from parent to child level).

But in this case, the parent component (settings) has blank path and API- Key path has path:api-keys path. Hence, angular router tries to load https://ritsrivastava01.github.io/Exads/setting-component-path/api-keys in the browser but it is shown as https://ritsrivastava01.github.io/Exads/api-keys. When we click on another menu, router tries to find that path inside api-keys and its fails to load that path. Hence, it throws the error (error in console).

Fix

If we provide the correct URL in the angular route then tabs will navigate properly.

e.g. if we first click on Go to Settings (top left first button) button then angular route will redirect to https://ritsrivastava01.github.io/Exads/settingsand then if we click on any tab, router will try to find that component inside the Settings component and it works fine!!!
